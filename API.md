# API Integration Guide\n\n## Architecture Overview\n\nThe app follows a clean architecture pattern:\n\n`\nReact Native App  \n       ↓\n    Screens\n       ↓\n  Custom Hooks (useDeviceData)\n       ↓\n   Services (DeviceService, RealtimeService, AuthService)\n       ↓\n  Supabase Client ← Real-Time Subscriptions\n       ↓\n  PostgreSQL Database\n`\n\n## Service Layers\n\n### 1. AuthService\n**Location:** `src/services/authService.ts`\n\n**Responsibilities:**\n- Email/password authentication\n- OTP phone verification\n- Session token management\n- Secure storage of credentials\n\n**Key Methods:**\n`typescript\nawait AuthService.loginWithEmail(email, password)\nawait AuthService.signupWithEmail(email, password, fullName)\nawait AuthService.verifyOTP(phone, token)\nawait AuthService.logout()\nawait AuthService.getCurrentSession()\nawait AuthService.refreshToken()\nawait AuthService.updateProfile(fullName, avatarUrl)\n`\n\n### 2. DeviceService\n**Location:** `src/services/deviceService.ts`\n\n**Methods:**\n`typescript\n// Fetch device data\nawait DeviceService.getUserDevices(userId)\nawait DeviceService.getDevice(deviceId)\nawait DeviceService.getCommunityDevices(communityId)\nawait DeviceService.getDeviceEvents(deviceId, limit)\nawait DeviceService.getDeviceOutages(deviceId, limit)\nawait DeviceService.getActiveOutages(deviceId)\n\n// Search & filter\nawait DeviceService.searchDevices(query)\n\n// Update status\nawait DeviceService.updateDeviceStatus(deviceId, status)\n`\n\n### 3. CommunityService\n**Location:** `src/services/deviceService.ts`\n\n**Methods:**\n`typescript\nawait CommunityService.getCommunities()\nawait CommunityService.getCommunity(communityId)\nawait CommunityService.getCommunityStats(communityId)\nawait CommunityService.getAdminCommunities(userId)\n`\n\n### 4. AnalyticsService\n**Location:** `src/services/deviceService.ts`\n\n**Methods:**\n`typescript\nawait AnalyticsService.getDailyStats(deviceId, startDate, endDate)\nawait AnalyticsService.getWeeklyStats(deviceId)\nawait AnalyticsService.getMonthlyStats(deviceId)\nawait AnalyticsService.getInsightsSummary(deviceId)\n`\n\n### 5. RealtimeService\n**Location:** `src/services/realtimeService.ts`\n\n**Real-Time Subscriptions:**\n`typescript\n// Subscribe to specific device events\nRealtimeService.subscribeToDeviceEvents(deviceId, callback)\n\n// Subscribe to all user device changes\nRealtimeService.subscribeToUserDevices(userId, callback)\n\n// Subscribe to device status\nRealtimeService.subscribeToDeviceStatus(deviceId, callback)\n\n// Subscribe to community devices\nRealtimeService.subscribeToCommunityDevices(communityId, callback)\n\n// Cleanup\nRealtimeService.unsubscribe(subscriptionId)\nRealtimeService.unsubscribeAll()\n`\n\n---\n\n## Data Flow Examples\n\n### Example 1: User Login\n`\n1. User enters email/password on AuthScreen\n2. AuthService.loginWithEmail() called\n3. Credentials sent to Supabase Auth\n4. Session token returned\n5. Token stored in secure storage\n6. Auth state updates → Navigation to Feed\n7. FeedScreen mounts → useUserDevices hook fires\n8. DeviceService.getUserDevices() fetches from Supabase\n9. RealtimeService.subscribeToUserDevices() starts listening\n10. Devices render on screen\n`\n\n### Example 2: Real-Time Status Update\n`\n1. IoT device sends event to MQTT: devices/IOT-0001/status {\"status\": \"OFF\"}\n2. Backend service receives event\n3. Backend writes to events table: {device_id, status: \"OFF\", timestamp}\n4. Supabase broadcasts change via Real-Time\n5. RealtimeService callback triggered\n6. React state updates\n7. FeedScreen re-renders with new status \"OFF\"\n8. Live duration counter starts\n`\n\n### Example 3: Fetch Community Stats\n`\n1. CommunitiesScreen loads\n2. useCommunities() hook fetches all communities\n3. For each community, useCommunityStats() called\n4. CommunityService.getCommunityStats() executes:\n   - Query devices table for community\n   - Query outages table for devices\n   - Calculate: uptime%, device count, outage count\n5. Stats render as CommunityItem components\n6. RealtimeService subscriptions watch for changes\n7. Stats auto-update if devices change\n`\n\n---\n\n## Supabase Integration\n\n### Authentication\n`typescript\n// Initialize\nimport { supabase } from './config/supabase'\n\n// Sign up\nconst { data, error } = await supabase.auth.signUp({\n  email: \"user@example.com\",\n  password: \"password123\",\n  options: {\n    data: {\n      full_name: \"John Doe\",\n      role: \"user\"\n    }\n  }\n})\n\n// Sign in\nconst { data, error } = await supabase.auth.signInWithPassword({\n  email: \"user@example.com\",\n  password: \"password123\"\n})\n\n// Sign out\nawait supabase.auth.signOut()\n`\n\n### Database Queries\n`typescript\n// Select\nconst { data, error } = await supabase\n  .from('devices')\n  .select('*')\n  .eq('owner_user_id', userId)\n\n// Insert\nconst { data, error } = await supabase\n  .from('outages')\n  .insert({\n    device_id: deviceId,\n    started_at: new Date().toISOString(),\n    is_active: true\n  })\n\n// Update\nconst { data, error } = await supabase\n  .from('devices')\n  .update({ status: 'ON' })\n  .eq('id', deviceId)\n\n// Complex queries\nconst { data } = await supabase\n  .from('devices')\n  .select('*, events(*), outages(*)')\n  .eq('community_id', communityId)\n`\n\n### Real-Time Subscriptions\n``typescript\nconst channel = supabase\n  .channel(`events:device_id=eq.${deviceId}`)\n  .on(\n    'postgres_changes',\n    {\n      event: '*',  // Listen to all changes (INSERT, UPDATE, DELETE)\n      schema: 'public',\n      table: 'events',\n      filter: `device_id=eq.${deviceId}`\n    },\n    (payload) => {\n      console.log('New event:', payload.new)\n      // Update React state\n    }\n  )\n  .subscribe()\n\n// Unsubscribe\nsupabase.removeChannel(channel)\n``\n\n---\n\n## Backend Service Integration\n\n### MQTT → Supabase Pipeline\n\nThe backend should implement:\n\n1. **MQTT Listener**\n`javascript\nconst mqtt = require('mqtt');\nconst client = mqtt.connect('mqtt://broker.url');\n\nclient.subscribe('devices/+/status', (err) => {\n  if (err) console.error(err);\n});\n\nclient.on('message', (topic, message) => {\n  // topic: devices/IOT-0001/status\n  // message: {\"status\": \"ON\"}|{\"status\": \"OFF\"}\n  handleDeviceEvent(topic, message);\n});\n`\n\n2. **Event Processing**\n`javascript\nasync function handleDeviceEvent(topic, messageBuf) {\n  const [, deviceId, ] = topic.split('/');\n  const { status } = JSON.parse(messageBuf.toString());\n  const timestamp = new Date().toISOString();\n\n  // Write to events table\n  await supabase\n    .from('events')\n    .insert({\n      device_id: deviceId,\n      status: status,\n      timestamp: timestamp\n    });\n\n  // Update device last_seen\n  await supabase\n    .from('devices')\n    .update({\n      status: status,\n      last_seen: timestamp\n    })\n    .eq('device_id', deviceId);\n\n  // Handle outage logic\n  if (status === 'OFF') {\n    // Start outage\n    const activeOutage = await getActiveOutage(deviceId);\n    if (!activeOutage) {\n      await supabase.from('outages').insert({\n        device_id: deviceId,\n        started_at: timestamp,\n        is_active: true\n      });\n    }\n  } else if (status === 'ON') {\n    // End outage\n    const activeOutage = await getActiveOutage(deviceId);\n    if (activeOutage) {\n      const duration = calculateDuration(activeOutage.started_at, timestamp);\n      await supabase\n        .from('outages')\n        .update({\n          ended_at: timestamp,\n          duration_minutes: duration,\n          is_active: false\n        })\n        .eq('id', activeOutage.id);\n    }\n  }\n}\n`\n\n3. **Offline Detection Job**\n`javascript\n// Run every 5 minutes\nsetInterval(async () => {\n  const fifteenMinutesAgo = new Date(Date.now() - 15 * 60 * 1000);\n  \n  const { data: devices } = await supabase\n    .from('devices')\n    .select('*')\n    .lt('last_seen', fifteenMinutesAgo.toISOString())\n    .neq('status', 'OFFLINE');\n\n  for (const device of devices || []) {\n    await supabase\n      .from('devices')\n      .update({ status: 'OFFLINE' })\n      .eq('id', device.id);\n  }\n}, 5 * 60 * 1000);\n`\n\n---\n\n## Error Handling\n\nAll services follow this pattern:\n\n`typescript\ninterface ApiResponse<T> {\n  success: boolean;\n  data?: T;\n `devices-${userId}`;\n  \n  if (cache.has(cacheKey)) {\n    return cache.get(cacheKey);\n  }\n  \n  const data = await DeviceService.getUserDevices(userId);\n  cache.set(cacheKey, data);\n  \n  // Clear after 5 minutes\n  setTimeout(() => cache.delete(cacheKey), 5 * 60 * 1000);\n  \n  return data;\n};\n``\n\n### Pagination Example\n`typescript\n// For large event lists\nawait supabase\n
