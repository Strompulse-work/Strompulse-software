# QUICKSTART.md - Get Running in 5 Minutes\n\n## Prerequisites Installed?\n- ✅ Node.js 16+\n- ✅ npm or yarn\n- ✅ Expo CLI installed globally\n- ✅ Supabase account\n\n## Step 1: Clone & Install (2 min)\n\n`bash\ncd ibadan-power-app\nnpm install\nnpx expo install\n`\n\n## Step 2: Supabase Setup (2 min)\n\n1. **Create Supabase Project**\n - Go to https://app.supabase.com\n - Click \"New Project\"\n - Name it \"ibadan-power-dev\"\n - Save your Project URL and Anon Key\n\n2. **Copy Environment Variables**\n `bash\n   cp .env.example .env\n   `\n \n3. **Edit `.env` file** with your Supabase credentials:\n `env\n   EXPO_PUBLIC_SUPABASE_URL=https://your-project.supabase.co\n   EXPO_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here\n   EXPO_PUBLIC_GOOGLE_MAPS_API_KEY=your-google-maps-key\n   `\n\n## Step 3: Create Database Tables (1 min)\n\n1. Go to Supabase Dashboard → SQL Editor\n2. Copy-paste the schema from `SCHEMA.md`\n3. Execute it\n\n## Step 4: Enable Real-Time (30 sec)\n\n1. Go to Supabase Dashboard → Database → Replication\n2. Toggle ON for: `devices`, `events`, `outages`, `communities`\n\n## Step 5: Start Dev Server (30 sec)\n\n`bash\nnpm start\n`\n\nThen choose:\n- **Android**: Press `a`\n- **iOS**: Press `i`\n- **Web**: Press `w`\n\n---\n\n## Testing the App\n\n### Demo Login (No Account Needed)\n`\nEmail: demo@ibadanpower.com\nPassword: demo123456\n`\n\n### Create Test Account\n`\nEmail: your-email@example.com\nPassword: any-password-6-chars-min\n`\n\n### Add Test Device\n\n1. In Supabase Dashboard → SQL Editor:\n `sql\n   INSERT INTO devices (device_id, owner_user_id, address, latitude, longitude, status, last_seen)\n   VALUES (\n     'IOT-0001',\n     'USER_ID_HERE',  -- Get from Auth users table\n     '10 Awolowo Road, Ibadan',\n     7.3775,\n     3.9465,\n     'ON',\n     now()\n   );\n   `\n\n2. Refresh app → See device on Feed & Map\n\n### Simulate Power Outage\n\n`sql\n-- In Supabase SQL Editor\nINSERT INTO events (device_id, status, timestamp)\nVALUES ('IOT-0001', 'OFF', now());\n\nUPDATE devices SET status = 'OFF', last_seen = now()\nWHERE device_id = 'IOT-0001';\n`\n\nWatch the app update in real-time! 🚀\n\n---\n\n## File Structure Guide\n\n`\nsrc/\n├── screens/         ← 5 main screens\n├── services/        ← Supabase API calls\n├── hooks/           ← Data fetching logic\n├── components/      ← Reusable UI\n├── types/           ← TypeScript types\n├── navigation/      ← Screen routing\n├── utils/           ← Helper functions\n├── config/          ← Supabase setup\n└── styles/          ← Colors, spacing\n`\n\n---\n\n## Common Tasks\n\n### Add a New Screen\n\n1. Create file: `src/screens/NewScreen.tsx`\n2. Add to RootNavigator.tsx\n3. Create route in app.json plugins\n\n### Add a Real-Time Feature\n\n`typescript\nimport RealtimeService from '../services/realtimeService'\n\n// In your screen/component\nuseEffect(() => {\n  const subId = RealtimeService.subscribeToDeviceEvents(\n    deviceId,\n    (payload) => {\n      console.log('Device event updated:', payload.new)\n      // Update UI\n    }\n  )\n  \n  return () => RealtimeService.unsubscribe(subId)\n}, [deviceId])\n`\n\n### Fetch Data with Hook\n\n`typescript\nimport { useUserDevices } from '../hooks/useDeviceData'\n\nconst { devices, loading, error } = useUserDevices(userId)\n\nif (loading) return <Loading />\nif (error) return <ErrorMessage message={error} />\nreturn <FlatList data={devices} ... />\n`\n\n---\n\n## Debugging Tips\n\n### Enable
